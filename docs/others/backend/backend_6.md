##　 uri, url, urn

### URI（上位概念）

- インターネット上のリソースを識別するすべての文字列
- URL と URN を両方含む

### URL（位置ベース）

```
https://naver.com/news/article/123
https://youtube.com/watch?v=abc123
ftp://files.example.com/document.pdf
```

### URN（名前ベース）

```
urn:isbn:9788932473901        (本のISBN)
urn:uuid:12345678-1234-5678   (固有識別子)
urn:ietf:rfc:3986             (RFC文書番号)
```

## ipv4

### IP の形

```
<!-- a -->
形式: X.Y.Z.W
ネットワーク: X (最初の数字のみ)
ホスト: Y.Z.W (残りの3つ)

例: 10.1.2.3
- ネットワーク: 10
- ホスト: 1.2.3

<!-- b -->
形式: X.Y.Z.W
ネットワーク: X.Y (前の2つの数字)
ホスト: Z.W (後ろの2つ)

例: 172.16.1.100
- ネットワーク: 172.16
- ホスト: 1.100

<!-- c -->
形式: X.Y.Z.W
ネットワーク: X.Y.Z (前の3つの数字)
ホスト: W (最後の1つ)

例: 192.168.1.100
- ネットワーク: 192.168.1
- ホスト: 100
```

### 公開 IP

- 世界中で唯一無二 (絶対に重複不可)
- インターネットから直接アクセス可能
- 国際機関が厳格に管理
- ISP(通信会社)から割り当て
- 費用が発生

###　私設 IP

- ラウターが付与
- 世界中で重複使用可能
- インターネットから直接アクセス不可
- 各組織が自由に使用
- 無料で使用可能
- NAT を通してのみインターネット接続

- 範囲

```
Aクラス: 10.0.0.0 ~ 10.255.255.255
Bクラス: 172.16.0.0 ~ 172.31.255.255
Cクラス: 192.168.0.0 ~ 192.168.255.255 (家庭用)
```

###　 NAPT(Network Address Port Translation)

- 既存 NAT は１：１の公開 IP、私設 IP の対応 → 非効率的
- 以下のように port で解決

```
私設 IP      port  →  公開 IP        port
192.168.1.100:8000  →  203.123.45.67:8001
192.168.1.101:8000  →  203.123.45.67:8002
192.168.1.102:9000  →  203.123.45.67:8003
```
