# 基本知識\_16

**Last updated:** 2025-10-16

## 外部ライブラリのバグ対応戦略

1. 問題原因の特定

   - 原因の区分
     - 問題原因の特定
     - 特定コンポーネントのみの問題か確認
     - Git 履歴、GitHub イシュー参考

2. 即時対応案

   - 影響度分析
     - ユーザーへの影響把握
   - 一時的解決策
     - CSS オーバーライド
     - 代替ライブラリ使用
     - 問題部分のみ直接実装

3. コミュニケーション

   - ステークホルダーへの共有
     - 企画者、PO、QA に状況説明
     - 予想影響と対応計画伝達
     - 同僚開発者と解決策模索

4. 長期的解決

   - 公式チャネル活用
     - バグレポート提出
     - PR で直接貢献
   - フォーク＆修正
     - 緊急時フォークして修正
     - 変更事項の文書化必須

## PO,PMO の違い

- PM (プロジェクトマネージャー)

  - 個別プロジェクトを直接管理する人
  - 1 つのプロジェクトに集中
  - プロジェクトの計画・実行・完了を担当

- PMO (プロジェクトマネジメントオフィス)

  - 組織全体のプロジェクト管理を標準化・支援する部署
  - 複数のプロジェクトを総括
  - PM 達に方法論、ツール、テンプレートを提供
  - プロジェクト間のリソース調整

- 例え: PM が「選手」なら、PMO は「コーチ兼戦略チーム」です。
